1 ¿Qué es Next.js y por qué se dice que es el framework de React más popular? 
Es el framework de React para construir aplicaciones web, es el mas popular porque permite crear aplicaciones con cara y ojos que se renderizan en el servidor, tienen buen SEO, optimizaciones automáticas y permiten hacer rutas de APIs de forma muy fácil.

2 ¿Cuál es el comando exacto con npx para inicializar un proyecto de Next.js asegurando usar la última versión?
npx create-next-app@latest

3 ¿Cuál es el comando utilizado para levantar el entorno de desarrollo y en qué puerto se ejecuta por defecto? 
npm run dev, se ejecuta por defecto en el puerto 3000

4 ¿Qué función cumple el comando npm run build en un proyecto de Next.js? 
Crea una versión de la aplicación optimizada para producción, transforma el codigo para que sea rapido y eficiente antes de desplegarlo

5 Describe para que sirven las carpetas, public y app
La carpeta public guarda los archivos estaticos como imagenes (logos) que se sirven directamente al navegador
La carpeta app contiene toda la logica de la aplicación, los componentes y define el sistema de enrutamiento

6 ¿Que es un CCS MODULE y como se usan? 
Son archivos CSS que permiten que los estilos sean locales al componente ( evitan colisiones de nombres). Se usan creando un archivo .module.css e importandolo en el componente para aplicar las clases como si fueran propiedades de un objeto

7 ¿Que hace el componente  Layout.tsx ? 
Es un componente que sirve para compartir la interfaz (como menús o barras laterales) entre varias páginas, todo lo que pongamos en el layout se mantendra constante mientras se navega por las rutas hijas

8 ¿Que es el enrutamiento ?
Es la forma en que la aplicación decide que contenido mostrar dependiendo de la dirección (URL) que el usuario visita en el navegador

9 ¿Cual es la diferencia entre Image e img?
Image es el componente de Next.js que optimiza automaticamente el tamaño, evita el salto de diseño (layout shift) y carga las imagenes solo cuando entran en el visor (lazy loading)
Img es la etiqueta nativa de HTML que no tiene optimizaciones

10 ¿Que es el Lazy Load?
Es una técnica que hace que las imagenes u otros recursos solo se descarguen cuando el usuario hace scroll y estan a punto de aparecer en la pantalla ahorrando datos y tiempo de carga

11 Cual es el funcionamiento del siguiente codigo, en que momento se muestra cada imagen
El codigo usa clases de Tailwind para mostrar imágenes diferentes según el dispositivo
hidden md:block Oculta la imagen en móviles y la muestra solo en pantallas medianas o grandes
block md:hidden Muestra la imagen en móviles y la oculta en pantallas grandes

12 ¿Como funciona el enrutamiento en Next JS  ? 
Funciona mediante carpetas dentro de app, el nombre de la carpeta se convierte en la ruta de la URL, para que una carpeta sea una ruta accesible debe contener un archivo llamado page.tsx

13 ¿Que es una ruta en el contexto web y donde las hemos visto en la clase ? 
Es la parte de la URL que identifica una página específica ejemplo: /dashboard

14 ¿Que arrojo a MiduDev el localhost:300/dashboard, donde hemos visto esos errores en la clase ? 
Al principio arroja un error 404 porque la ruta no existe, vimos ese error cuando haciamos peticiones de nuestros endpoints en el postman

16 ¿Que  es el File System Routing?
Es el sistema donde la estructura de archivos y carpetas define automáticamente las rutas de la aplicación sin necesidad de configurar un archivo de rutas por separado

17 ¿Como funcionan las rutas anidadas?
Se crean poniendo una carpeta dentro de otra, por ejemplo una carpeta invoices dentro de dashboard crea la ruta /dashboard/invoices

18 ¿Investiga por que es necesario esto ?
Es necesario porque Next.js espera que cada archivo de página (page.tsx) exporte una función por defecto para saber qué es lo que tiene que renderizar en esa ruta

19 ¿Para que sirve el componente Link?
Sirve para navegar entre las páginas de la aplicación de forma rápida, realiza la navegación en el lado del cliente (Client-side navigation)

20 ¿Que se carga cuando usamos <a> y que cuando se usa <link> ?
<a>: Provoca una recarga completa de la página, se descargan todos los recursos de nuevo
<Link>: Solo descarga los datos necesarios para la nueva página sin recargar todo el sitio haciendo la transición casi instantánea

21 ¿Investiga que es un Hook?
Son funciones de React que permiten usar el estado y otras características del framework como el ciclo de vida dentro de componentes funcionales

22 ¿Que hooks usa Midu?
Midu usa principalmente el hook usePathname para detectar en qué ruta se encuentra el usuario y así resaltar el enlace activo en el menu

23 ¿Por que ocurre el siguiente error?
Ocurre porque se intenta usar un hook usePathname en un Server Component, los hooks solo funcionan en Client Components

24 ¿Es posible ejecutar los componentes de react en el cliente y el servidor?
Si Next.js permite mezclar ambos por defecto son de servidor pero pueden transformarlos a cliente cuando se necesita interactividad

25 ¿Que es el code splitting?
Es una técnica donde Next.js divide el código de la aplicación en paquetes pequeños por ruta
